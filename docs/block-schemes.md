# Блок-схемы программы

Ниже приведены 3 блок-схемы, оформленные стандартными типами блоков:

- `([Текст])` — начало/конец.
- `[/Текст/]` — ввод/вывод.
- `[Текст]` — процесс.
- `[[Текст]]` — предопределенный процесс.
- `{Текст?}` — проверка условия.
- `((N))` — соединитель.

## 1. Блок-схема работы программы

```mermaid
flowchart TD
    A([Начало]) --> B[/Запуск веб-приложения/]
    B --> C[Отображение главной страницы]
    C --> D{Пользователь авторизован?}

    D -- Нет --> E{Есть аккаунт?}
    E -- Да --> F[/Ввод логина и пароля/]
    E -- Нет --> G[/Ввод данных для регистрации/]

    F --> H[[Авторизация]]
    G --> I[[Регистрация пользователя]]
    I --> J{Регистрация успешна?}
    J -- Нет --> G
    J -- Да --> K[Сохранение токена и данных пользователя]

    H --> L{Данные верны?}
    L -- Нет --> F
    L -- Да --> K

    D -- Да --> M{Определение роли}
    K --> M

    M --> N{Роль = VOLUNTEER?}
    N -- Да --> O[Переход в кабинет волонтера]
    N -- Нет --> P{Роль = ORGANIZER?}

    P -- Да --> Q[Переход в кабинет организатора]
    P -- Нет --> R[Переход в панель координатора/админа]

    O --> S{Выбор раздела}
    S --> S1[Просмотр задач]
    S --> S2[Профиль и навыки]
    S --> S3[Мои заявки]
    S1 --> T{Выход из аккаунта?}
    S2 --> T
    S3 --> T

    Q --> U{Выбор раздела}
    U --> U1[Управление задачами]
    U --> U2[Заявки волонтеров]
    U --> U3[Матчинг кандидатов]
    U --> U4[Профиль организатора]
    U1 --> T
    U2 --> T
    U3 --> T
    U4 --> T

    R --> V{Выбор раздела}
    V --> V1[Панель координатора]
    V --> V2[Матчинг]
    V --> V3[Просмотр профилей]
    V1 --> T
    V2 --> T
    V3 --> T

    T -- Нет --> M
    T -- Да --> W[[Выход из системы]]
    W --> X([Конец])
```

## 2. Блок-схема процесса регистрации пользователя

```mermaid
flowchart TD
    A([Начало]) --> B[/Открытие формы регистрации/]
    B --> C[/Ввод логина, email, пароля и роли/]
    C --> D[[Проверка заполнения полей]]
    D --> E{Все поля заполнены корректно?}

    E -- Нет --> F[/Сообщение об ошибке/]
    F --> C

    E -- Да --> G[[Проверка уникальности логина]]
    G --> H{Логин свободен?}
    H -- Нет --> I[/Ошибка: логин уже занят/]
    I --> C

    H -- Да --> J[[Проверка уникальности email]]
    J --> K{Email свободен?}
    K -- Нет --> L[/Ошибка: email уже зарегистрирован/]
    L --> C

    K -- Да --> M[Определение роли пользователя]
    M --> N[Хеширование пароля]
    N --> O[Создание учетной записи]
    O --> P{Роль = VOLUNTEER?}

    P -- Да --> Q[Создание профиля волонтера]
    P -- Нет --> R[Пропуск создания профиля волонтера]

    Q --> S[Генерация JWT-токена]
    R --> S
    S --> T[Сохранение данных авторизации]
    T --> U[/Переход на главную страницу/]
    U --> V([Конец])
```

## 3. Блок-схема процесса подачи заявки на задачу

```mermaid
flowchart TD
    A([Начало]) --> B[/Открытие карточки задачи/]
    B --> C[/Нажатие кнопки "Откликнуться"/]
    C --> D[[Поиск профиля волонтера]]
    D --> E{Профиль волонтера найден?}

    E -- Нет --> F[/Ошибка: профиль не найден/]
    F --> Z([Конец])

    E -- Да --> G{Волонтер активен?}
    G -- Нет --> H[/Ошибка: неактивный волонтер не может подать заявку/]
    H --> Z

    G -- Да --> I[[Поиск выбранной задачи]]
    I --> J{Задача существует?}
    J -- Нет --> K[/Ошибка: задача не найдена/]
    K --> Z

    J -- Да --> L{Статус задачи OPEN?}
    L -- Нет --> M[/Ошибка: отклик возможен только на открытую задачу/]
    M --> Z

    L -- Да --> N[[Проверка предыдущих заявок]]
    N --> O{Есть активная заявка?}
    O -- Да --> P[/Ошибка: активная заявка уже существует/]
    P --> Z

    O -- Нет --> Q{Количество попыток < 2?}
    Q -- Нет --> R[/Ошибка: лимит заявок исчерпан/]
    R --> Z

    Q -- Да --> S[/Ввод комментария к заявке/]
    S --> T[Создание заявки со статусом PENDING]
    T --> U[Сохранение номера попытки]
    U --> V[Сохранение заявки в базе данных]
    V --> W[/Сообщение: заявка отправлена/]
    W --> Z([Конец])
```
